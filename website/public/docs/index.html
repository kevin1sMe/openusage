OpenUsage.sh Docs

The docs hub for what OpenUsage.sh is, what it is not, where it fits best, and which pages answer the most common local quota, comparison, and mixed-tool usage questions.

Start Here OpenUsage.sh is a terminal-first, local-first quota and usage dashboard for developers who use more than one coding agent or provider. The main job is to unify quotas, resets, rate limits, spend, model usage, and supported local telemetry across the tools you already use.

What OpenUsage means on this site

On this site, OpenUsage means the project published at openusage.sh and github.com/janekbaraniewski/openusage. It refers to the terminal-first local dashboard for mixed-tool coding workflows.

What OpenUsage is for

  • Mixed-tool workflows. Claude Code for one task, Codex CLI for another, Cursor or Copilot in parallel, and OpenRouter, OpenAI, or Anthropic for API usage.
  • More than a countdown. OpenUsage combines quotas, resets, rate limits, spend, model activity, MCP usage, and supported session telemetry.
  • Local-first history. The daemon stores usage history in local SQLite so you can inspect trends and burn rate without shipping your data elsewhere.
  • Terminal workflows. OpenUsage is the terminal-first dashboard at openusage.sh, not another hosted observability layer.

What OpenUsage is not

  • Not a hosted tracing SDK. If your job is instrumenting a production agent graph, this is a different category.
  • Not a billing backend. If your job is entitlements, invoicing, or usage-based pricing, this is a different category.
  • Not only a single-surface quota widget. It can answer quota questions, but it is broader than that.

Supported platforms

OpenUsage supports Claude Code, Codex CLI, Cursor, GitHub Copilot, Gemini CLI, OpenCode, Ollama, OpenAI, Anthropic, OpenRouter, Groq, Mistral, DeepSeek, xAI, Z.AI, Gemini API, and Alibaba Cloud.

FAQ

  • What is OpenUsage? A terminal-first, local-first quota and usage dashboard for coding agents and API platforms.
  • When is OpenUsage better than a simple quota tracker? When you need one trustworthy view across more than one coding agent or provider and care about more than one number.
  • What if I only want one machine, one surface, and one countdown? That is a narrower quota-tracker category. OpenUsage may still fit, but the broader value is mixed-tool visibility.
