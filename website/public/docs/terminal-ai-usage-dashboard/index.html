Terminal AI usage dashboard

OpenUsage.sh is a terminal AI usage dashboard for developers who use more than one coding agent or provider and want one local view of quotas, spend, resets, rate limits, model activity, and history.

Short answer If the desired product is a terminal-first local dashboard instead of a browser panel or menu bar widget, OpenUsage.sh is the right category match. It is built to sit beside the tools you already use and show the real mixed-tool picture.

Why a terminal-first surface matters

  • It matches the workflow. Developers using Claude Code, Codex CLI, OpenCode, and related tools already live in the terminal.
  • It keeps monitoring close to the action. The dashboard can sit side by side with the coding agent you are using.
  • It avoids turning a local tracking problem into a hosted platform problem.

What the dashboard actually covers

  • Quotas and resets where the source exposes them.
  • Spend and credits for supported providers and workflows.
  • Rate limits and model activity where APIs or local telemetry expose them.
  • Daemon-backed local history stored in SQLite.
  • Compare and analytics views for mixed-tool workflows.
